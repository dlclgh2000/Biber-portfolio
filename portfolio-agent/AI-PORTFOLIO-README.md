# 🎨 AI 포트폴리오 생성기

Claude Code와 통합되어 시니어 프로덕트 디자이너 수준의 전문적인 포트폴리오를 자동으로 생성하는 도구입니다.

## ✨ 특징

- **피그마 디자인 분석**: 피그마 링크를 제공하면 자동으로 디자인을 분석
- **AI 기반 작성**: Claude가 시니어 디자이너 관점에서 전문적인 케이스 스터디 작성
- **자동 HTML 생성**: 상세 페이지 HTML 자동 생성
- **완전 자동화**: 프로젝트 데이터 및 메인 페이지 자동 업데이트

## 🚀 빠른 시작

### 1단계: 프로젝트 정보 입력

```bash
node create-portfolio.mjs
```

다음 정보를 입력합니다:
- 프로젝트 ID
- 제목 및 부제목
- 피그마 링크 (선택)
- 프로젝트 설명
- 이미지 경로

### 2단계: Claude에게 요청

스크립트 실행 후 출력되는 메시지를 Claude Code에게 전달합니다.

```
portfolio-request-{project-id}.json 파일을 읽고,
시니어 프로덕트 디자이너 관점에서 전문적인 포트폴리오를 작성해주세요.
```

### 3단계: 자동 생성 완료

Claude가 자동으로:
1. 피그마 디자인 분석 (링크가 있는 경우)
2. 전문적인 케이스 스터디 작성
3. `detail-{project-id}.html` 생성
4. `projects-data.json` 업데이트
5. `index.html` 업데이트

## 📋 생성되는 포트폴리오 구조

### 1. 프로젝트 개요
프로젝트의 핵심 목표와 배경을 2-3문장으로 요약

### 2. 문제 정의 (Problem Statement)
- 해결하려는 사용자 문제
- 비즈니스 과제

### 3. 디자인 목표 (Design Goals)
프로젝트의 3-5가지 핵심 디자인 목표

### 4. 주요 기능 (Key Features)
- 핵심 기능 3-5개
- 각 기능이 문제를 해결하는 방법

### 5. 디자인 프로세스 (Design Process)
- 🔍 리서치
- 💡 아이디어 발산
- 🎨 프로토타이핑
- 🧪 테스트
- 🔄 반복 및 개선

### 6. 주요 디자인 결정 (Design Decisions)
- 3-4가지 중요한 디자인 결정
- 각 결정의 근거와 배경

### 7. 성과 및 임팩트 (Impact & Results)
- 📊 측정 가능한 성과
- 💬 사용자 피드백
- 💼 비즈니스 임팩트

### 8. 배운 점 (Learnings)
프로젝트를 통해 얻은 2-3가지 인사이트

## 📁 파일 구조

```
Biber-portfolio/
├── create-portfolio.mjs              # 메인 생성 스크립트
├── portfolio-request-{id}.json       # 프로젝트 정보 (자동 생성)
├── detail-{id}.html                  # 상세 페이지 (Claude가 생성)
├── projects-data.json                # 프로젝트 목록 (자동 업데이트)
├── index.html                        # 메인 페이지 (자동 업데이트)
└── AI-PORTFOLIO-README.md            # 이 문서
```

## 💡 팁

### 피그마 링크 활용
- 피그마 링크를 제공하면 Claude가 실제 디자인을 분석하여 더 구체적인 포트폴리오 작성
- UI 컴포넌트, 디자인 시스템, 인터랙션 등을 자동으로 파악

### 프로젝트 설명 작성 가이드
다음 내용을 포함하면 더 좋은 포트폴리오가 생성됩니다:
- **배경**: 왜 이 프로젝트를 시작했나요?
- **목표**: 무엇을 달성하려고 했나요?
- **타겟 유저**: 누구를 위한 제품인가요?
- **주요 기능**: 어떤 기능들이 있나요?
- **기술/도구**: 어떤 도구를 사용했나요?
- **성과**: 어떤 결과를 얻었나요? (정량적/정성적)

### 이미지 준비
생성 전 다음 이미지를 준비하세요:
- **썸네일**: 메인 페이지 그리드에 표시 (추천: 1200x800px)
- **히어로 이미지**: 상세 페이지 상단 (추천: 1600x900px)
- **슬라이더 이미지**: 메인 페이지 슬라이더용 (선택)

## 📞 문제 해결

**Q: 피그마 링크가 인식되지 않아요**
- 공개 링크인지 확인하세요
- 파일 링크가 아닌 프레임 링크를 사용하세요

**Q: 생성된 포트폴리오가 마음에 들지 않아요**
- `portfolio-request-{id}.json`에서 설명을 더 구체적으로 수정
- Claude에게 "더 전문적으로", "더 구체적으로" 등의 요청 추가

**Q: HTML이 깨졌어요**
- `detail-{id}.html` 파일을 삭제하고 다시 생성
- 백업 파일이 자동 생성되므로 걱정 없음

2. **피그마 활용**: 가능하면 피그마 링크 제공
3. **이미지 품질**: 고해상도 이미지 사용
4. **일관성**: 비슷한 포맷의 이미지 사용
5. **검토**: 생성 후 내용 검토 및 필요시 수정
